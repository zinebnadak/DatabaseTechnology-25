Your Workflow Overview

Here‚Äôs the standard workflow when working with .sql files in a project:

Step	Tool	Purpose
1	PyCharm	Write or edit .sql files (like setup.sql, insert_data.sql, etc.)
2	DBeaver	Connect to your database (mydatabase.db) and run SQL scripts to create tables, insert data, or query data
3	GitHub	Version control your .sql files so you can share or collaborate
4	Optional	Python scripts or apps can also connect to mydatabase.db to use the data

üí° Key point: PyCharm edits, DBeaver runs, GitHub stores. That‚Äôs the typical separation.

Run Your SQL Scripts in DBeaver

In the Database Navigator, right-click your database ‚Üí SQL Editor ‚Üí New SQL Script.

In the editor that opens:

Either copy-paste your SQL from setup.sql, or

Click Open File and select setup.sql.

Click the green ‚ñ∂Ô∏è Execute button (or press ‚åò + Enter) to run all the SQL commands.

You can now see the tables you created in the Database Navigator ‚Üí Tables section.

To check data:

Right-click a table (e.g., users) ‚Üí View Data ‚Üí All Rows.

Now you can visually confirm your database contents.

3Ô∏è‚É£ Keep Your .sql Files in GitHub

Even though DBeaver edits the database, you do not commit the .db file (optional, usually only for small projects). Instead:

Make sure setup.sql is in your GitHub repo (it should be in your PyCharm project folder).

In PyCharm Terminal:

git status       # See untracked files
git add setup.sql
git commit -m "Add initial SQL setup"
git push


Now your .sql script is versioned. Anyone else can run it on their own SQLite database and get the same tables/data.

This is how developers share SQL: the script files, not always the actual .db database.

4Ô∏è‚É£ Editing / Updating

If you want to add new tables or data:

Edit or create a new .sql file in PyCharm (e.g., insert_more_data.sql).

Run it in DBeaver on your database.

Commit the new .sql file to GitHub.

This way, your SQL workflow is fully version controlled and reproducible.

5Ô∏è‚É£ Optional: Automate Running SQL from Python

Later, you can even write a Python script that connects to your SQLite database and runs your .sql files automatically using the sqlite3 library:

import sqlite3

conn = sqlite3.connect("mydatabase.db")
cursor = conn.cursor()

with open("setup.sql") as f:
    sql_script = f.read()

cursor.executescript(sql_script)
conn.commit()
conn.close()


This is how many developers integrate SQL into Python projects while keeping .sql scripts in GitHub.
